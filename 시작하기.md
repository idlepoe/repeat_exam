화면 구조 (모바일 대응 appbar가 표시, 이력이 있는 화면의 경우 왼쪽 상단에 뒤로가기가 표시) 
css 는 최대한 사용하지 말것.

1.exam_type_list 시험 리스트
[제빵기능사]
[제과기능사]

초기처리
 exam_type_list.json 을 읽어 화면에 표시

2.exam_session_list 출시회차

[뒤로가기]

제빵기능사
2002-01-27
2002-04-07

초기처리
 선택된 exam_type에 해당하는 exam_session_list.json 의 데이터를 읽어 화면에 표시


3.question 문제화면
----------------
[뒤로가기] {exam_type}
[{subject}] aA(폰트크기)
{question_number}.{question_text}

{question_image_url}

for loop choices
 {choices.no}.{text}  (correct_answer와 동일할 경우 배경을 빨간색으로 폰트는 흰색으로 표시)

[이전]  <변경>(누르면 이전 다음의 버튼 위치가 변경)  [다음]
----------------

초기처리
 해당 exam_type, exam_session에 해당하는 json을 읽어서 1개 question씩 표시
 storage에 저장되어 있는 exam_type과 exam_session에 해당하는 question_number가 있다면 해당 number로 이동

[뒤로가기] exam_session_list로 이동
[이전] 이전 question_number로 이동
[다음] 다음 question_number로 이동
 현재 exam_type과 exam_session의 question_number 를 storage에 저장함.
 마지막 question_number 일 경우 '다음 회차로 이동할까요? 질문하고 '취소' '목록' '다음 회차' 로 표시
 
 해당 다이얼로그의 [목록] 또는 [다음 회차] 버튼을 눌렀을 때 동작
 해당 exam_type과 exam_session에 대한 count를 +1 처리하여 storage에 저장.
 exam_type과 exam_session에 해당하는 question_number는 삭제.
